Methodology & Data Provenance

How the content on this site is produced, what's verified, and how to check anything you're unsure about.

TL;DR

Most charts and numeric frequencies currently on this site are educational illustrations, not solver-verified outputs. They capture the correct qualitative ideas (which positions open wider, which hands are 3-bet bluff candidates, how frequency scales with position), but specific percentages and individual cell classifications have not been cross-checked against a real GTO solver run.

Data provenance tiers

As verified data is added, individual charts and claims will carry one of three badges:

  • Solver-verified Generated from a documented solver run (TexasSolver, PioSolver, etc.) with input parameters available in the site repo.
  • Published Taken from a third-party published source (GTO Wizard, Upswing charts, a named book) with citation.
  • Unverified Educational illustration only — directionally correct but not solver-checked.

Any chart or frequency without an explicit badge should be treated as unverified by default.

How to verify a chart yourself

  1. Free preflop: GTO Wizard exposes a free tier with solver-verified preflop ranges for common stack depths. Compare cell-by-cell against the chart you see here.
  2. Run a solver: TexasSolver is free and open source. Feed it the same spot (stacks, positions, bet sizes, ranges) and compare frequencies. Differences larger than a few percent are worth investigating.
  3. Cross-read known authorities: Modern Poker Theory (Acevedo) for frameworks; Play Optimal Poker (Saulsberry) for accessible single-spot reasoning.

Contributing verified data

This site's data layer lives in data/ranges/*.json. Each file follows the schema below. If you solve a spot yourself (or export a range from GTO Wizard), drop it in and the site will render it with the appropriate provenance badge. 

{
  "id": "6max-utg-rfi-100bb",
  "title": "6-max UTG RFI (100bb)",
  "source": {
    "tier": "verified",           // "verified" | "published" | "unverified"
    "solver": "TexasSolver 0.2.0", // required if tier=verified
    "params": {
      "stacks_bb": 100,
      "rake_pct": 5,
      "rake_cap_bb": 3,
      "open_size_bb": 2.5
    },
    "citation": null,              // required if tier=published
    "run_date": "2026-04-19",
    "run_file": "solver/runs/6max-utg-rfi-100bb.json"
  },
  "grid": {
    // 169 entries keyed by canonical hand notation
    "AA":  { "action": "raise", "freq": 1.0 },
    "AKs": { "action": "raise", "freq": 1.0 },
    "AJo": { "action": "raise", "freq": 0.75 },
    "ATo": { "action": "mixed", "freq": 0.35 }
    // ...
  },
  "notes": "Solved on a single node; ignores flat-call from BB at open size 2.5x."
}

Why not just replace everything with verified data now?

Building a full solver pipeline that covers every chart on the site is a real project — solving runs take minutes to hours per spot, and the full tree (preflop + postflop across positions, stack depths, and bet sizes) is thousands of spots. We're building the plumbing first so verified data can land incrementally without breaking the existing pages. See the repo for the data pipeline and open spots.

What we will not do

  • Copy proprietary charts from paid products (GTO Wizard Premium, PioSolver libraries, paid courses) into the site. Free-tier charts with permissive terms only.
  • Represent unverified output as verified. If a chart doesn't carry an explicit verified or published badge, it's an illustration.
